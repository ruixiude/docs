# RAF Android SDK - Core 介绍

Core 划分为以下数个模块：

  • bus - 总线服务
  • common - 通用工具包
  • crash - 异常跟踪服务（支持多方服务实现同时使用）
  • database - 数据库服务（支持替换服务实现）
  • log - 日志服务（支持替换服务实现）
  • network - 网络工具包
  • octopus - 依赖注入服务
  • oss - 对象存储服务（支持替换服务实现）
  • router - 路由服务
  • statistics - 数据统计服务（支持多方服务实现同时使用）

# 特性

每个模块均可单独使用，建议根据实际开发需求进行组合使用。

  • 部分模块的服务实现支持选配
  • crash 与 statistics 模块支持同时使用多个服务实现

# 代码风格

基于以下约束

  1. Android Lint
  2. Plugin: Alibaba Java Guidelines

# 测试覆盖

按照代码运行场景，分以下情况

  • Java - 使用 JUnit 测试框架编写单元测试用例
  • Android - 脱机环境下使用 robolectric 测试框架，需要测试 NDK 模块则需要使用 Android Support Test 或者 AndroidX Test 测试框架

代码覆盖率建议在 80% 以上

# 使用示例

新增模块建议附加相关的使用示例，项目中的 app module 就是用于放置演示代码与演示场景。

# 发版计划

划分为：

  • 维护 - 每周发布一个版本，统一在每周周二发布
  • 模块调整 - 根据调整是否符合预期进行发布
  • 架构调整 - 根据现有技术以及使用反馈，进行预估，当需要进行架构调整，则会设立若干目标，当目标均已实现，则进行发布

# 协作规则

# 项目成员

  1. 按照版本计划，向对应的分支提交代码
  2. bug 修复和功能开发不能在同一个分支上提交，bug 需要独立基于 main 分支开启新分支进行处理
  3. 在版本计划内的功能无法达到预期效果，需要推延到下一个版本中。

# 非项目成员

  1. Fork 本项目代码
  2. 完成代码编辑并通过单元测试
  3. 向本项目发出 MR，并附带详尽的修改说明
